# Speaker Notes — How I AI

**Session format:** 45 min total
- ~5 min opening (before slides)
- ~6 min The Why (slides 1–6)
- ~8 min The Tools (slides 7–13)
- ~6 min Explore Each Tool (slides 14–18)
- ~13 min Cursor Getting Started (slides 19–23 + video demo)
- ~7 min Q&A / closing

**Audience:** Mostly skeptical or overwhelmed SEs. Some power users. Some who don't know where to start.
**Tone:** Teammate sharing what they figured out. Not a mandate. Not a keynote.
**Presenting URL to share:** `https://jarteaga-sf.github.io/se-ai-toolkit/`

---

## Opening (~5 min, before slides)

Before you advance to slide 1, set the room:

> "I'm not here to tell you AI is the future — you've heard that. I'm here to show you what I've actually been using and what I've seen teammates use to get real things done faster. Some of it will apply to you right now. Some of it is worth bookmarking for later. By the end of this you'll have a link to come back to anytime."

If the room feels skeptical:

> "If you're skeptical — good. That means you've already seen a lot of AI hype that didn't pan out. What I'm going to show you is different. These are tools SEs are using today, in real deals."

## Explore Each Tool — Slides 14–18

These slides are meant to be quick — about 60–90 seconds each. The goal is that the audience leaves knowing what each tool is and what distinguishes it. You're not demoing — you're giving them enough to know which one is theirs.

**On screen:** Logo + name + tagline + 3 examples on the right.

## Cursor Getting Started — Slides 19–23

---

### Slide 19 — Four Habits That Make the Difference
**On screen:** Four habit cards — Start with Tab / Use Agent Mode / Give It Context / Review Before Accepting.

> "Before I show you the steps to get set up, I want to give you the four habits that separate people who get real value from Cursor from people who install it and never use it."

**Start with Tab:**
> "Don't think about it. Just start typing. Tab to accept, keep typing to ignore. Zero effort."

**Agent Mode:**
> "Open the chat, switch to Agent. It can read your files, write code, run commands — all from one prompt. This is where the real leverage is."

**Give It Context:**
> "Type @ in the chat to pull in files, docs, or your whole project. The more it knows, the better the output. Two sentences about the customer makes every suggestion more relevant."

**Review Before Accepting:**
> "Always read what it changed. It shows you the diff before you accept. It's fast — it's not always right. Read it."

---

### Slide 20 — License → Install → Connect
**On screen:** Three-phase step flow.

> "Three steps. License, install, connect. If you've used VS Code before, this takes about 20 minutes. If you haven't, closer to 30. Either way, you're up and running today."

## Q&A — Common Objections

**"I don't have time to learn another tool."**
> "Cursor's Tab autocomplete requires zero learning. Start typing, press Tab when the suggestion is right. You're already using it."

**"My prospects won't care how I built it."**
> "They won't care about the tool. They'll care that the demo looks like their business — their company name, their problem, their terminology. The tool is just how you get there in minutes instead of days."

**"What about hallucinations / AI being wrong?"**
> "Treat every output as a first draft. Review before you show a customer. Fast doesn't mean finished. The mental model: AI does the first 80%, you do the last 20%. That 80% is what used to take all day."

**"Is this approved? Is our data safe?"**
> "Cursor has Privacy Mode — enable it during setup, nothing leaves your machine. Saleo overlays data in the browser, never touches your real org. For Claude Code and MeshMesh, connect only to demo orgs. Never a customer org, never production."

**"MeshMesh sounds great — when can I get access?"**
> "It's in pilot right now. Watch for the dedicated session invite — that's when we'll walk through access requests together."

## Screen Studio Recording — Click Path

Record this before the session. Target: 5–6 minutes, tight highlight reel. The presenter narrates over it live.

**What to prepare before recording:**
- Demo project open in Cursor (a Salesforce LWC component — something with a button, a data table, a title)
- Privacy Mode already enabled (you can show the setting briefly)
- Chat panel closed — start clean
- Salesforce extension active

**The 6 scenes:**

**Scene 1 — Open Cursor (0:00–0:20)**
Show the VS Code-familiar interface. File explorer on the left, editor center, terminal at the bottom.
*Talk track: "If you've used VS Code, this is home. If you haven't — this is easier."*

**Scene 2 — Tab autocomplete (0:20–1:15)**
Start typing in a JavaScript file. Ghost text appears. Accept with Tab x3. Do it naturally, not staged.
*Talk track: "Zero setup. It sees what you're trying to do before you finish. Tab to accept, keep typing to ignore."*

**Scene 3 — Cmd+K quick edit (1:15–2:30)**
Select a button or heading element. Press Cmd+K. Type: `"Make this button Salesforce blue (#0176D3), rounded corners, subtle shadow"`. Show the diff in the editor. Accept.
*Talk track: "Cmd+K. Select code, describe the change in plain English, see exactly what it'll do before you accept. You're not writing code — you're describing an outcome."*

**Scene 4 — Cmd+L — ask about code (2:30–3:30)**
Open chat (Cmd+L). Type: `"What does this component do? I'm new to this project."` Show the clear, plain-language explanation.
*Talk track: "Inherited a project you didn't write? Just ask. Understand in seconds what would take 30 minutes of reading."*

**Scene 5 — Agent mode (3:30–5:00)**
Switch to Agent in the chat panel. Type: `"Add a loading spinner that shows while the data fetches. Keep the Salesforce styling."` Watch it read multiple files, plan the approach, make changes. Show the file diff summary. Accept.
*Talk track: "Agent mode. One prompt, it read three files, planned the approach, made all the changes. I reviewed and accepted. That's the whole workflow."*

**Scene 6 — The result (5:00–5:30)**
Show the running component — spinner appears, data loads, styling is correct.
*Talk track: "Start to finish — one goal, plain English, working code. That's what getting started looks like."*
